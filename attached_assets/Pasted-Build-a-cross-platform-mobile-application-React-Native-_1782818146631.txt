Build a cross-platform mobile application (React Native / Expo preferred) for **QuickFleet Workforce Solutions** — an Indian B2B company offering workforce supply, IT infrastructure support, and CCTV/security services.
The app must mirror the existing public marketing website and connect to the SAME Spring Boot backend API already in use. Do NOT rebuild the admin panel. This is a customer-facing mobile app only.
---
## 1. PRODUCT OVERVIEW
**App name:** QuickFleet  
**Tagline:** Reliable Workforce & Technical Solutions for Growing Businesses  
**Target users:** Business owners, HR managers, facility heads, and job seekers in India  
**Core value:** Browse services, view company info, submit quote inquiries, browse jobs, apply with resume, view gallery, contact company
**Backend base URL (configurable via env):**
`http://localhost:8080/api/v1` (dev) — use env variable `API_BASE_URL`
All API responses follow this wrapper:
```json
{
  "success": true,
  "message": "...",
  "data": { ... }
}
Always unwrap data from responses. Paginated lists use Spring Page format:

{
  "content": [...],
  "totalElements": 0,
  "totalPages": 0
}
No authentication required for public app users.

{
  "name": "required",
  "email": "required",
  "phone": "optional",
  "subject": "required",
  "message": "required"
}
Form fields:

Full Name*
Company (optional — prepend to message)
Phone*
Email*
Service Required* (dropdown from services list)
City* (append to subject as "{service} — {city}")
Message*
Validation: email format, phone 7–15 digits, max lengths per backend.

Success: "Inquiry received. We'll respond within one business day."

8. OFFLINE & ERROR HANDLING
Cache API responses locally (AsyncStorage) for 5 minutes
Show static fallback content from bundled JSON if API fails (use the default QuickFleet copy from the website)
Network error banner: "Unable to load latest content. Showing saved version."
Form submissions require network — show clear error if offline
Loading states on every screen
Empty states: "No jobs available", "No gallery images yet"
9. TECHNICAL REQUIREMENTS

Navigation: React Navigation (bottom tabs + stack)
File upload: expo-document-picker for resume on careers apply
Env config: API_BASE_URL in .env
Deep linking: Optional — quickfleet://contact, quickfleet://careers/{slug}
